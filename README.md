# Image Viewer

BMP / PNG / JPEG / TIFF / GIF 対応の軽量画像ビューア。  
フォルダ内の画像をキーボードで連続閲覧できるほか、矩形選択による色計測・ブレ計測・解像感計測、および2画像の詳細比較解析機能を備えます。

## 必要環境

| 依存 | バージョン |
|------|-----------|
| Python | 3.10 以上 |
| PyQt5 | 5.15 以上 |
| numpy | 1.21 以上 |
| opencv-python | 4.5 以上 |

## インストール (Windows)

```bat
install.bat
```

`py.exe` (Python Launcher) 経由で依存パッケージをインストールします。  
Python 本体は https://www.python.org/ からインストールしてください。

## 起動方法

### Windows（タスクバーから）

`image_viewer.vbs` のショートカットを作成し、タスクバーにピン留めして使います。  
ダブルクリックでコンソール非表示のウィンドウとして起動します。

### Linux / WSL (WSLg)

```bash
./launch.sh [画像ファイルまたはフォルダパス]
```

## 基本操作

| 操作 | 動作 |
|------|------|
| `←` / `→` | 前/次の画像 |
| `Ctrl+O` | フォルダを開く |
| `Ctrl+D` | 画像比較ウィンドウを開く |
| `Ctrl++` / `Ctrl+-` | ズームイン/アウト |
| `Ctrl+0` | フィット表示 |
| マウスホイール | ズーム (Ctrl+ホイール) / スクロール |
| 中ボタンドラッグ | パン |
| 左ドラッグ | 矩形選択（計測） |
| `Esc` | 選択解除 / 比較ウィンドウを閉じる |

### ドラッグ＆ドロップ

Windows Explorer からファイルまたはフォルダをウィンドウにドロップして開けます。  
WSLg 環境でも動作します。

---

## 計測機能

矩形を左ドラッグで選択すると、ステータスバーに以下の値が表示されます。

- **RGB / Lab 平均・標準偏差** — 選択領域の色統計
- **ブレ量** — Laplacian 分散によるブレ指標（値が小さいほどブレている）
- **解像感** — Sobel エッジ強度による鮮鋭度指標

---

## 画像比較ウィンドウ (`Ctrl+D`)

2枚の画像を並べて詳細解析するウィンドウです。  
**左ペイン = 画像A、中央ペイン = 画像B、右ペイン = 解析結果**

3ペインのズーム・パン・スクロールは連動します（Ctrl+ホイール / 中ボタンドラッグ）。

### 共通仕様

- **位置ズレ補正**: 「位置ズレ」タブで推定後、「この補正を解析に適用」ボタンを押すと以降の全解析でズレ補正済みのBを使用します。
- **リニア変換**: WB・カラー/彩度タブはsRGB逆変換（IEC 61966-2-1）でリニア光量に変換してから比率・色差を計算します。

---

### タブ詳細

#### 位置ズレ

位相相関（FFT）＋テンプレートマッチングで画像B がA に対してどれだけズレているかを推定します。

| 項目 | 説明 |
|------|------|
| 推定方法 | 位相相関で大まかなズレを検出 → 中央パッチでテンプレートマッチングにより精度検証 |
| グリッド解析 | 4×4 グリッドで局所的なズレを計測し、一様なズレか局所変形かを判定 |
| 補正適用 | 「この補正を解析に適用」で以降の全タブにアフィン補正を適用 |

**解析結果（第3ペイン）**: 輝度差グレーマップ（白=B明るい、黒=B暗い、灰=同等）

---

#### WB（ホワイトバランス）

リニア光量でのR/G/B比率を全体と4×4空間ブロック別に集計し、WBのズレ方向と空間的均一性を解析します。

**グラフ（棒グラフ）**

| 系列 | 意味 |
|------|------|
| R比率（赤棒） | 各ブロックでの img-B ÷ img-A のR値リニア比 |
| G比率（緑棒） | 同 G値リニア比（基準。均等なら全ブロック≒1.0） |
| B比率（青棒） | 同 B値リニア比 |

X軸は「全体」＋「(行,列)」形式のブロック名。  
G比率が1.0から大きく外れている場合は露出差（トーンカーブ差）の影響があります。

**解析結果（第3ペイン）**: 各チャンネル差分マップ  
RGB3チャンネルそれぞれで「img-B ÷ img-A」の偏りを表示。  
白=img-B過剰、黒=img-A過剰、灰=同等。

---

#### トーンカーブ

輝度帯（16分割）ごとに img-B / img-A の輝度比を集計し、明るさの変換特性（トーンカーブ）の差を可視化します。

**グラフ（折れ線）**

- X軸: img-A の輝度値（0〜255）
- Y軸: B/A 輝度比
- 破線: 基準ライン（1.0 = 同一）
- 暗部（左）と明部（右）で比率が異なる → トーンカーブのズレあり

**解析結果（第3ペイン）**: 輝度差グレーマップ  
白=img-Bが明るい、黒=img-Bが暗い、灰=同等。

---

#### カラー/彩度

リニア変換後にCIE Lab色空間へ変換し、色相帯別のΔL\*/Δa\*/Δb\*/ΔC\*を解析します。

**グラフ（棒グラフ）**

| 系列 | 意味 |
|------|------|
| ΔL\*（明度） | img-Bの明度 − img-Aの明度（+ = B明るい） |
| Δa\*（赤+/緑−） | + = img-Bが赤寄り、− = 緑寄り |
| Δb\*（黄+/青−） | + = img-Bが黄寄り、− = 青寄り |
| ΔC\*（彩度） | + = img-Bの彩度が高い |

X軸は色相帯（全体 / 赤 / 黄 / 緑 / シアン / 青 / マゼンタ）。  
色相帯ごとにΔ値が異なる場合、カラーマトリクスの違いを示します。

**解析結果（第3ペイン）**: Lab色差カラーマップ  
赤=img-Bが赤/黄寄り、緑=img-Bが緑寄り、青=img-Bが青寄り、灰=差なし。

---

#### ノイズ

局所標準偏差（7×7ウィンドウ）をノイズ量の代理指標として、エッジ以外の平坦部のみで輝度帯別のノイズ量を比較します。

**グラフ（折れ線 2系列）**

- X軸: 輝度値（0〜255）
- Y軸: 局所標準偏差の中央値
- A系列（青）: img-A のノイズ量
- B系列（赤）: img-B のノイズ量

B系列がA系列より低い → img-B のNRが強い。  
暗部だけ差がある → 暗部限定のNR処理の可能性。

**解析結果（第3ペイン）**: ノイズ差グレーマップ  
白=img-Aがノイジー（img-BのNR強い）、黒=img-Bがノイジー、灰=同等。

---

#### オーバーレイ

| モード | 説明 |
|--------|------|
| 輝度差 | 白=img-B明るい、黒=img-B暗い、灰=同等（スケール自動） |
| ブレンド | スライダーで A/B の合成比率を調整 |
| エッジ重ね合わせ | シアン=Aのエッジ、赤=Bのエッジ、白=一致 |
